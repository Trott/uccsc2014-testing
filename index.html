We will walk you through this process from the basic setup to our more advanced tips and tricks. The best part is all of our work is open source so you can take home both an understanding of these technologies and some real-world examples to start from.

Behat

... or selenium if you happen to be a masochist

We wrote our tests in behat its a behaviour testing system for PHP. This is the only peice of the presentation that depends on PHP, everythgin else can be done in any languate.

Using behat create browser tests which cover all of the critical behaviours of your application and add new tests whenever you find a bug.

Github

... or something else if you happen to be a masochist

Github is hands down the best colaborative code hosting environment. They are constantly working on new features and its position as the home of great open source software means that everything else integrates seamlessly.

Travis-CI

We use travis to do automated tests of every commit, branch and pull request. This ensures that it is nearly impossible for new code or changes to break existing functionality. Travis can also do a ton of other stuff, like packaging or publishing your builds.

Sauce Labs

We picked Sauce because it tests on a ton of browsers, integrates with travis, and its free for open source.
